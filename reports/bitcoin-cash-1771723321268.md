## 한줄 요약
Bitcoin Cash Node 28.0.1은 기존 28.0.0에 비해선택적 버그수정과 성능개선이 포함된 패치 릴리즈다. 주요 변경사항은 RPC 인터페이스에 두 개의 새로운 필드가 추가되고, 네트워크 프로토콜에서 타임스탬프 검증 로직이 개선되며, 메모리 풀과 블록 처리 성능이 향상된 점이다. 이 릴리즈는 기존 28.0.0에서 발생한 작은 버그를 수정하고, 대규모 블록 처리 시 성능을 높여 노드 부하를 줄인다. 28.0.1은 메인넷에 대한 큰 기능 변경이 없으므로, 업그레이드가 필수는 아니지만 보안 및 성능 향상을 위해 권장된다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- - `getpeerinfo` RPC에 `bip152_hb_to`, `bip152_hb_from` 필드 추가
- - `getnetworkinfo` RPC에 `connections_in`, `connections_out` 키 추가
- - 네트워크 프로토콜에서 타임스탬프 무시 로직(버그수정)
- - 메모리 풀 `removeForBlock()` 성능 개선 및 `CHECK_NONFATAL` 사용
- - ABLA 체크 로직 단순화

## RPC/API 영향

- {"method":"getnetworkinfo","added_fields":["connections_in","connections_out"]}

## 아카이브 노드 영향
- 아카이브 노드가 새 필드를 무시해도 정상 동작한다. 1) RPC 호출 시 추가된 필드가 반환되지만, 아카이브 노드는 이 정보를 사용하지 않으므로 별도 영향 없음. 2) 네트워크 타임스탬프 무시 로직은 아카이브 노드가 과거된 타임스탬프를 가진 블록을 수신할 때 무시하도록 하므로, 데이터 일관성에 영향 없음. 3) 메모리 풀 성능 개선은 아카이브 노드가 대규모 블록을 저장할 때 CPU/메모리 사용량이 감소한다. 4) ABLA 체크 단순화는 부팅 시 소요되는 시간은 줄이지만, `-check-abla` 옵션을 사용하면 기존 체크 로직과 동일한 동작을 유지할 수 있다.

## 운영 액션 아이템
- 1. 노드 소프트웨어를 최신 버전(28.0.1) 으로 업그레이드 한다.
- 2. RPC 호출 시 새 필드를 무시하거나, 클라이언트가 이 필드에 의존하지 않게 한다.
- 3. 네트워크 타임스탬프 무시 로직이 적용된 노드가 연결되는지 확인한다. 4. 메모리 풀 성능 개선이 적용된 경우, `mempool` 설정을 재검토한다. 5. ABLA 체크 단순화 옵션(`-check-abla`) 사용 여부를 검토한다.

## 마이그레이션 체크리스트
- 1. 현재 노드가 28.0.0 버전인지 확인한다.
- 2. 백업을 수행한 뒤, 28.0.1 로 업그레이드한다.
- 3. RPC 호출 시 새 필드를 지원하는지 테스트한다. 4. 네트워크 타임스탬프 무시 로직이 정상 동작하는지 테스트한다. 5. 메모리 풀 성능 개선에 따른 설정(`mempool`)을 재검토한다. 6. ABLA 체크 단순화 옵션(`-check-abla`) 사용 여부를 결정한다.
