## 한줄 요약
이 릴리즈(버전: GreatVoyage‑v4.8.1)는 TRON 프로토콜의 핵심 기능과 인프라를 개선한 버전으로, JSON‑RPC 예외 처리 로직이 추가되고 DB/스토리지 레이어가 리팩토링되었습니다. 주요 변경사항은 RPC 예외 클래스 추가, WriteOptionsWrapper 삭제, RocksDB/LevelDB 구현 개선 등이다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 외부 API 엔드포인트/버전 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- * JSON‑RPC 예외 처리 로직 추가: JsonRpcExceedLimitException, JsonRpcException, JsonRpcInternalException, JsonRpcInvalidParamsException, JsonRpcInvalidRequestException, JsonRpcMethodNotFoundException, JsonRpcTooManyResultException 등 7개 파일 추가/수정
- * WriteOptionsWrapper 클래스 삭제 및 대체 로직 도입(레벨DB/로켓스DB)
- * RocksDbSettings, RocksDbDataSourceImpl 등 스토리지 레이어 리팩터링
- * VM 관련 코드(EneryCost, OperationActions 등) 개선

## 아카이브 노드 영향
- 1. WriteOptionsWrapper 삭제로 인한 DB 접근 방식 변경이 아카이브 노드의 데이터 저장/읽기 로직에 영향을 줄 수 있다. 2. RocksDB 설정이 업데이트되었으므로, 기존 아카이브 노드가 새 버전에서 재구성 필요

## 운영 액션 아이템
- 1. RPC 예외 로직이 추가되었으므로, 클라이언트(예: 지갑)에서 예외 처리 로직을 업데이트한다. 2. RocksDB/LevelDB 설정 파일(`RocksDbSettings.java`)을 최신 버전으로 교체한다. 3. 노드 재시작 시 DB 리인덱싱이 필요하면 수행한다.

## 마이그레이션 체크리스트
- 1. 현재 노드가 최신 버전(4.8.1)인지 확인한다. 2. RPC 예외 로직이 추가되었으므로, 클라이언트/지갑에 적용된 예외 처리 로직이 있는지 점검한다. 3. RocksDB/LevelDB 설정을 최신으로 교체하고, WriteOptionsWrapper 삭제에 따른 DB 접근 로직이 정상인지 테스트한다. 4. 아카이브 노드가 재구성 필요 시, `TronDatabase` 및 `TronStoreWithRevoking`를 재빌드한다. 5. 테스트넷에서 먼저 시범 운영 후 메인넷에 배포한다.
