## 한줄 요약
이 릴리즈는 주로 geth 구성 옵션과 Genesis 설정에 대한 내부 구조를 정리하고, ECBP1100 관련 필드를 새로 명명한 `OverrideECBP1100` 으로 교체함으로써, 기존의 big.Int 기반 설정을 단순화하고 명시적 옵션 처리 방식을 도입한다. 주요 변경 사항은 CLI 플래그, JSON‑TOML 직렬화/역직렬화 로직, 그리고 Genesis 디코더에 대한 분기처리이다. 이로 인한 RPC/JSON‑RPC 인터페이스는 변동이 없으나, 설정 파일과 실행 시 옵션에 대한 호환성 검증이 필요하다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

- 주요 변경점 없음

- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드가 Genesis 설정에 따라 ECBP1100 옵션을 사용한다면, 새 필드(`OverrideECBP1100`) 로 교체되었으나, 기존 설정 파일에 `ecbp1100` 값이 남아 있으면 파싱 실패 가능. 따라서 아카이브 노드 설정 파일을 점검 필요.

## 운영 액션 아이템
- 기존 `--ecbp1100` 옵션을 사용 중이라면, `--override-ecbp1100` 으로 교체.
- Genesis 파일에 `OverrideECBP1100` 값이 없으면, 새 필드가 nil 이므로 기본값 사용.
- 설정 파일(`config.toml`) 에서 `ecbp1100` 키를 삭제하고, 필요 시 `override_ecbp1100 = <value>` 으로 교체.
- 배포 전 테스트 환경에서 새 옵션이 정상 동작하는지 확인.

## 마이그레이션 체크리스트
- 기존 설정 파일에서 `ecbp1100` 키를 찾아서 `override_ecbp1100` 으로 교체.
- CLI 스크립트/옵션이 `--ecbp1100` 인 경우, `--override-ecbp1100` 으로 교체.
- Genesis 디코더가 두 버전간 차이를 인식하도록 설정되었는지 테스트.
- 아카이브 노드가 새 필드를 인식하는지 확인.

## 위험/주의 사항
- 기존 `ECBP1100` 옵션이 big.Int 로 정의되어 있었으나, 새 필드 `OverrideECBP1100`(uint64) 로 교체되면서 기존 스크립트/설정 파일에서 타입 변환이 필요하다.
- Genesis 디코더가 core‑geth 와 go‑ethereum 두 버전간 차이를 인식하도록 변경되었으나, 잘못된 경우 Genesis 파일이 파싱 실패할 가능성 존재.
- 옵션 이름 변경(`ECBP1100` → `OverrideECBP1100`) 으로 인한 스크립트/환경설정(예: `--ecbp1100`) 가 동작하지 않을 수 있다.

## 운영 메모
- 옵션 이름이 바뀌면서 CLI 플래그(`--ecbp1100`) 가 `OverrideECBP1100` 으로 바뀜. 기존 스크립트에서 이 옵션을 사용하고 있다면, `--ecbp1100` 를 `--override-ecbp1100` 으로 교체해야 함.
- Genesis 디코더가 두 버전간 차이를 인식하도록 변경되었으나, `params/types/genesisT` 에서 `OverrideECBP1100` 으로 설정이 반영되도록 확인 필요.
- JSON‑TOML 직렬화/역직렬화 로직이 `OverrideECBP1100` 으로 교체되었으나, 기존 설정 파일에 `ecbp1100 = ...` 이 있다면 파싱 실패 가능.
