## 한줄 요약
v2.6.0 릴리즈는 주로 메타데이터와 부트노드 목록, 그리고 포크 타임스탬프를 업데이트한 버전입니다. 주요 변경사항은 1) genesis 서브모듈을 v2.2.0으로 교체, 2) snake8Time 타임스탬프 추가, 3) 새 부트노드 주소 삽입이며, 버전 번호를 2.5.1 → 2.6.0 으로 바꾴는 것입니다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드가 새 genesis 서브모듈을 가져와야 함. 현재는 메타데이터만 바뀜이므로 재인덱스 필요 없음
- fork 타임스탬프가 바뀜에 따라, 시점이 변동되면 재인덱스 필요할 수 있으나 현재는 0 차이

## 운영 액션 아이템
- config/embedded/chiliz.json 파일을 최신 버전으로 교체
- genesis 서브모듈 업데이트 후 재배포
- 부트노드 리스트에 새 주소가 반영되었는지 확인
- 버전 번호 2.6.0 으로 업데이트된 바이너리 재배포
- 노드가 새 fork 타임스탬프를 인식하도록 설정 파일 재로드

## 마이그레이션 체크리스트
- 새 부트노드 주소가 정상 동작하는지 테스트
- genesis 서브모듈이 올바르게 반영되었는지 확인
- fork 타임스탬프가 현재 시점에 맞게 설정되었는지 검증
- 버전 번호 변경에 따른 버전 체크 로직(예: go‑modules, docker‑image tag) 업데이트
- 아카이브 노드가 새 config를 반영하도록 재시작

## 위험/주의 사항
- 새로운 부트노드가 네트워크에 정상적으로 연결되는지 확인 필요
- genesis 서브모듈 교체가 기존 노드에 미치는 영향은 최소하지만, 구버전과의 호환성 검증이 필요
- fork 타임스탬프 변경으로 인한 시점 차이로 인한 잠재적 동기화 이슈

## 운영 메모
- 버전 번호가 5 → 6 으로 바뀜 것이므로, 기존 RPC 버전 체크 로직이 있다면 업데이트 필요
- 부트노드 추가는 단순 문자열 삽입이므로 RPC/JSON‑RPC 인터페이스에는 영향 없음
- genesis 서브모듈 교체는 메타데이터만 바뀜; 실제 체인 데이터에는 변동 없음
