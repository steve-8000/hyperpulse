## 한줄 요약
XDC 프로토콜 v1.0.1 릴리즈는 주로 Docker 이미지 버전 업그레이드와 RPC/WS API에 XDPoS 모듈을 추가한 점이 핵심이다. 이 변경은 노드 운영자가 기존 설정에 대한 작은 조정만으로 적용 가능하지만, 신규 API가 도입되면서 일부 클라이언트와 스크립트에 대한 호환성 검증이 필요하다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- docker‑compose 이미지 버전: v2.2.4 → v2.3.0
- start‑node.sh에 --rpcapi, --wsapi 옵션에 XDPoS 추가
- deployment‑generator 버전 1.0.0 → 1.0.1

## RPC/API 영향
- XDPoS API가 RPC/WS 엔드포인트에 추가되었습니다. 예: --rpcapi db,eth,debug,net,shh,txpool,personal,web3,XDPoS

## 아카이브 노드 영향
- 아카이브 노드가 XDPoS 관련 데이터를 저장하도록 설정을 추가해야 할 수 있음. 현재는 별도 변경이 없으나, XDPoS 관련 인덱스가 추가될 경우 스토리지 용량이 증가할 수 있다.

## 운영 액션 아이템
- 1. Docker 이미지 버전 업데이트( v2.3.0 ) 2. start‑node.sh 에서 --rpcapi, --wsapi 옵션에 XDPoS 추가 여부 확인 3. upgrade.sh 에서 이미지 pull 명령이 바뀜지 확인 4. 노드 재시작 전 백업 및 테스트

## 마이그레이션 체크리스트
- 1. 현재 이미지가 v2.3.0 인지 확인 2. XDPoS API가 정상 동작하는지 테스트(예: web3, net 등) 3. 배포 스크립트에 버전 문자열이 올바르게 반영되는지 검증 4. 아카이브 노드가 XDPoS 데이터를 저장하도록 설정 여부 확인
