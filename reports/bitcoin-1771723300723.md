## 한줄 요약
v28.4rc1은 현재 개발 중인 28.x 시리즈의 첫 번째 릴리스 후보이며, 주요 변경 사항은 CI/빌드 스크립트 정리와 일부 문서·테스트 개선에 집중돼 있다. 28.x 시리즈의 핵심 기능(트랜잭션, 블록체인, RPC 등)은 기존 28.3과 크게 변동이 없으나 몇몇 작은 API/문서 수정이 포함된다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드가 기존 시드를 사용하지 않으므로, 현재 아카이브 노드는 정상 동작한다. 별도 재설정 필요 없음.

## 운영 액션 아이템
- CI 빌드 스크립트가 변경되었으므로, `autogen.sh`와 `configure` 옵션(`--with-incompatible-bdb`)을 사용해 빌드
- 테스트 스크립트(`ci/test/00_setup_env_*.sh`)를 최신으로 교체
- 아카이브 노드가 `-DWITH_ZMQ=ON` 옵션을 사용해 빌드되었으나, ZMQ가 필요 없으면 `-DWITH_ZMQ=OFF` 옵션으로 재빌드
- RPC 문서 정정이 반영된지 확인

## 마이그레이션 체크리스트
- 1. 빌드 환경(리눅/윈도우)에서 `autogen.sh` 실행 후 `make -j$(nproc)`
- 2. 테스트 스크립트(`ci/test/00_setup_env_*.sh`)를 최신으로 교체
- 3. `getpeerinfo` 문서가 업데이트되었으니, 문서 검증
- 4. 아카이브 노드가 `chainparams.cpp` 변경을 반영했는지 확인
- 5. RPC API가 변동이 없으므로 별도 마이그레이션 필요 없음

## 운영 메모

- 지금까지의 변경은 체인 파라미터(예: seed)와 관련된 파일이 삭제되었으나, 실제 아카이브 노드 동작에는 영향이 없다. 기존 `chainparams.cpp`에서 불안정 시드가 제거되었으나, 이는 테스트용이므로 운영에 영향 없음.
