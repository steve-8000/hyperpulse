## 한줄 요약
Aptos v1.41.5 릴리즈는 주로 VM과 메모리풀, 스토리지 설정에 대한 성능 및 보안 관련 조정이 포함된다. 핵심 변경 사항은 비동형 타입 검사 활성화, 메모리풀 용량 확대, RocksDB 스레드 수 조정 등이다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 트랜잭션 검증 시 추가 비용 발생
- 메모리풀 용량 증가
- RocksDB 백그라운드 스레드 수 증가

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- RocksDB low_priority_background_threads 증가로 인한 백그라운드 compaction 지연 가능. 아카이브 노드는 주로 읽기 전용이므로 큰 영향은 없으나, 대량 데이터 삽입 시 지연 주의 모니터링 필요.
- async_runtime_checks가 true 로 기본 설정되었으므로, 아카이브 노드에서도 트랜잭션 검증 시 추가 비용 발생. 대량 데이터 삽입 시 CPU 부하가 증가할 수 있다.

## 운영 액션 아이템
- async_runtime_checks를 true 로 설정한 상태에서 성능 벤치마크 수행
- orderless_txn_capacity_per_user를 5,000 으로 설정한 뒤 메모리 사용량 모니터링
- RocksDB low_priority_background_threads를 4 로 설정 후 스토리지 성능 모니터링
- 비동형 타입 검사 활성화에 따른 CPU/메모리 사용량 점검

## 마이그레이션 체크리스트
- config/src/config/execution_config.rs에서 async_runtime_checks를 true 로 설정했는지 확인
- config/src/config/mempool_config.rs에서 orderless_txn_capacity_per_user가 5,000 으로 설정됐는지 확인
- config/src/config/storage_config.rs에서 low_priority_background_threads가 4 로 설정됐는지 확인
- 노드 재배포 전 성능 벤치마크(특히 async_runtime_checks 활성화 시) 수행
- RocksDB 설정 변경이 스토리지 성능에 미치는 영향 테스트

- 위험 요소 없음

## 운영 메모
- 비동형 타입 검사 도입은 보안 강화에 기여하지만, 실행 지연을 유발할 수 있다. 테스트 환경에서 성능 벤치마크를 재실행해야 함.
- 메모리풀 용량 확대는 메모리 사용량을 5배까지 늘릴 수 있으므로, 운영 환경에서 메모리 할당 조정 필요.
- RocksDB 스레드 수 증가가 백그라운드 작업에 미치는 영향은 실제 운영 환경에서 모니터링 필요.
