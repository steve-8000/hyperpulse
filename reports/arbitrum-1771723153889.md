## 한줄 요약
v3.9.6‑rc.2 릴리즈는 주로 내부 구조 재배치와 오래된 기능 제거, 그리고 새로운 컨센서스 버전(51.1) 도입을 중심으로 진행되었습니다. 주요 변경 사항은 ① 컨센서스 버전 업그레이드와 관련 Docker 이미지 설정, ② 레거시(legacy) 블롭/에이프 포인트 제거, ③ 디버그·테스트 인프라 구조 변경(아카이브/리포지터리 재배치) 등입니다. 이 릴리즈는 RPC 인터페이스에 직접적인 변동이 적지만, 블롭 관련 엔드포인트와 아카이브 노드 저장소 설정에 영향을 미칩니다. ④ “EnableChunkedStore” 기본값이 false 로 바뀐 점은 저장소 성능에 영향을 줄 수 있습니다.

## 운영자 중요 체크
- 외부 API 엔드포인트/버전 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 레거시 블롭 사이드카 엔드포인트가 Fusaka 도입으로 사라짐
- 레거시 버전 0 블롭 프로브 지원이 사라짐

## 아카이브 노드 영향
- `EnableChunkedStore` 기본값이 false 로 바뀐 점으로 인한 저장소 성능 변동. 아카이브 노드가 새 설정을 반영하도록 `config.toml` 혹은 환경변수 조정 필요
- 포스트‑Fusaka 로컬 디스크 리더 추가로 인한 파일 시스템 접근 패턴이 바뀜. 아카이브 노드가 `arbitrator/target` 경로를 사용하도록 캐시 설정을 업데이트해야 함

## 운영 액션 아이템
- Docker 이미지 재빌드
- CI 파이프라인 재구성
- RPC 클라이언트 코드 업데이트
- 아카이브 노드 재배포

## 마이그레이션 체크리스트
- 노드가 51.1을 지원하도록 `config.toml` 에 `consensus = "v51.1"` 설정
- RPC 클라이언트가 `eth_getBlobSidecars` 혹은 `eth_getBlobProofV0` 를 호출하지 않도록 테스트
- `EnableChunkedStore` 기본값이 false 인 점을 반영해 저장소를 재구성
- `arbitrator/target` 경로를 반영한 Dockerfile 빌드가 정상 동작하는지 확인

## 위험/주의 사항
- 레거시 블롭/에이프 엔드포인트(legacy blob sidecars, legacy blob proof) 제거로 인한 기존 클라이언트 호환성 이슈
- 컨센서스 버전 51.1 도입으로 인한 내부 로직(예: `assume-valid`, `split-val`) 변동이 기존 노드에 미치는 영향
- Dockerfile 경로 및 캐시 설정 변경으로 인한 CI/배포 파이프라인 재구성 필요
- `EnableChunkedStore` 기본값 변경으로 인한 저장소(스토리지) 재구성 필요

## 근거(Evidence)

- Remove legacy blob_sidecars endpoint support now that Fusaka has launched
- Store blobs locally post fusaka + disk reader

## 운영 메모
- 레거시 엔드포인트가 사라지면서 기존 RPC 호출이 실패할 가능성 존재. 클라이언트는 `eth_getBlobSidecars` 같은 호출을 제거하거나 대체 로직으로 교체해야 함
- 컨센서스 51.1 도입은 `assume-valid` 로직이 추가되었으며, 이는 노드가 비정상 블록을 무시하도록 하는 옵션이므로 운영자는 `--assume-valid` 플래그를 검토해야 함
- Dockerfile 및 CI 스크립트가 `arbitrator/` 디렉터리 구조를 반영하도록 재구성 필요
- 아카이브 노드가 `arbitrator/target` 경로를 사용하도록 캐시 설정을 업데이트해야 함
