## 한줄 요약
이번 커밋은 주로 네트워크 상수정과 종속성 버전업그에 초점을 두고 있다. 핵심 변경은 API에서 FujiID를 TahoeID 로 매핑하고, Genesis 설정에서 txfee 패키지와 validatorfee 의 import 경로를 정리하며, go.mod/​go.sum 에서 coreth 와 mock 의 버전을 올린다. 이로 인한 RPC/JSON‑RPC 인터페이스는 크게 변하지 않으나, GetTxFee 로직이 네트워크 ID에 따라 달라 바뀜. 아카이브 노드가 새 파라미터를 로딩할 때 LocalParams 의 TxFeeConfig 가 바뀜 점을 주의미한다.


주요 변경 포인트:
1. constants.FujiID 가 TahoeID 로 교체.
2. genesis_local.go 에서 txfee 패키지 삭제, validatorfee.Config 로 교체.
3. go.mod 에서 coreth v0.14.1‑rc.1 으로 업그레이드.
4. go.sum 에서 golang.org/x/mod, x/net 버전 조정.

이 변경은 API 로직과 Genesis 파라미터에 영향을 주며, 특히 아카이브 노드가 새 파라미터를 로딩할 때 동작에 영향을 미칠 수 있다.


위험: 1) 네트워크 ID 매핑 오류로 인한 GetTxFee 응답이 잘못될 수 있다. 2) Genesis 파라미터 변경으로 인해 기존 노드가 새 파라미터를 인식 못할 수 있다. 3) go.mod/​go.sum 의 버전업이 의존재 패키지 충돌 가능성.


위험을 최소화하기 위해서는 테스트를 통해 RPC 응답과 Genesis 파라미터가 정상인지 검증하고, 아카이브 노드 재시작 시 파라미터를 재검토해야 한다.


위험을 감지하고, 운영자는 아래와 같은 조치를 취해야 한다.
