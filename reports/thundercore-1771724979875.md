# ThunderCore 운영 리포트

## 한줄 요약
본 릴리즈는 ThunderCore 프로토콜 버전 r4.3.8 로 업그레이드하며, 이미지 버전과 하드포크 설정을 업데이트한다. 주요 변경사항은 환경설정 파일(.env.example)과 mainnet 하드포크 설정(configs-template/mainnet/hardfork.yaml)에 대한 값 교체 및 신규 파라미터 추가이다. 이로 인한 RPC/API/JSON‑RPC 인터페이스 변화는 없으며, 노드 운영 측면에서는 주로 하드포크 파라미터 조정에 따른 동작 변화가 존재한다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 하드포크 파라미터가 바뀜 경우, 아카이브 노드는 최신 버전으로 재시작해야 하며, 이전 세션 데이터와 호환성 검증이 필요
- 새로운 파라미터가 적용되면, 블록체인 스냅샷 저장 시 추가 메타데이터가 필요할 수 있다. 스토리지 용량 조정 검토

## 운영 액션 아이템
- 이미지 레지스트리에서 r4.3.8 이미지가 존재하는지 확인
- 노드 설정 파일(.env.example) 업데이트 후 재배포 스크립트 실행
- 하드포크 파라미터가 적용된 세션(12089/12090)에서 동작 테스트 수행
- 아카이브 노드 재시작 시, 최신 하드포크 파라미터를 반영하도록 설정 파일 업데이트

## 마이그레이션 체크리스트
- 이미지 버전 확인 및 컨테이너 재배포
- 하드포크 파라미터(`electVersion`, `vaultVersion`) 적용 여부 검증
- JSON‑RPC 엔드포인트 테스트(변경 없음)
- 아카이브 노드 재시작 시 스냅샷 호환성 테스트
- 모니터링 지표(블록체인 상태, baseFee 등) 재설정

## 위험/주의 사항
- 새로운 `electVersion` 및 `vaultVersion` 설정이 적용되면서, 특정 세션(12089/12090)에서 동작 방식이 달라질 수 있다. 이로 인한 노드 간 불일치 가능성
- `TESTNET_IMAGE_VERSION`, `MAINNET_IMAGE_VERSION` 값이 r4.3.8 로 바뀜다. 기존 이미지가 아직 배포되지 않은 경우, 노드 재배포 시 다운타임 발생 가능

## 근거(Evidence)
- 근거 데이터 없음

## 운영 메모
- 하드포크 파라미터가 추가되었으나, RPC/API 레벨에서는 직접적인 변경이 없다. 따라서 JSON‑RPC 엔드포인트는 그대로 사용 가능
- 다만, `electVersion` 과 `vaultVersion` 값이 바뀜 경우, 특정 세션에서 동작 방식이 달라질 수 있으므로, 노드가 해당 파라미터를 읽어 적용하도록 설정이 필요
- 이미지 버전 변경은 컨테이너/배포 스크립트에 반영되므로, 배포 시 이미지 레지스트리 확인 필수
