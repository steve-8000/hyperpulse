## 한줄 요약
Linea 5.0‑rc2에서는 트레이스 생성과 블록 컨플레이트 기능이 확장되었으며, 기존 Tx‑Compressor 및 Blob‑Compressor 모듈이 제거되었습니다. 이로 인해 RPC 엔드포인트가 추가되고, 트레이스 파일 생성 로직이 변경되었으며, 압축 기능이 사라졌습니다. 이 변화는 노드 운영에 있어야 할 설정 조정과 API 호출 업데이트를 요구합니다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 외부 API 엔드포인트/버전 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 새 RPC 엔드포인트 "linea_generateVirtualBlockConflatedTracesToFileV1" 추가
- TxCompressor 및 Blob‑Compressor 모듈 삭제(파일 33개 삭제)
- plugin 옵션 "--plugin-linea-limitless-enabled" 및 "--plugin-linea-module-limit-file-path" 설정 변경
- trace‑generation 로직에 대한 테스트 추가

## RPC/API 영향
- linea_generateVirtualBlockConflatedTracesToFileV1

## 아카이브 노드 영향
- 아카이브 노드가 "linea_generateVirtualBlockConflatedTracesToFileV1" 호출을 지원하도록 설정 필요
- 트레이스 파일 생성 경로가 변경될 수 있어 스토리지 정책 재검토 필요

## 운영 액션 아이템
- 1. TracesGeneratorJsonRpcClientV2를 최신 버전으로 교체
- 2. plugin 옵션 "--plugin-linea-limitless-enabled" 과 "--plugin-linea-module-limit-file-path" 를 검토 후 적용
- 3. TxCompressor/Blob‑Compressor 관련 코드가 삭제되었으니, 필요 시 별도 구현
- 4. 아카이브 노드에 트레이스 파일 수집 스크립트 업데이트

## 마이그레이션 체크리스트
- 1. RPC 엔드포인트가 추가되었는지 확인
- 2. 클라이언트 라이브러를 최신 버전으로 교체
- 3. plugin 옵션 설정 파일 검토 및 적용
- 4. 트레이스 파일 경로/이름 규칙 확인
- 5. TxCompressor 기능이 필요하다면 별도 구현 검토
