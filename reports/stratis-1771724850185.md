## 한줄 요약
이 릴리즈는 주로 Go 버전 업그레이드(1.22 → 1.23)와 몇 가지 내부 코드 정리(메모리 사이즈 관련 삭제, IsOnCurve 검증 추가)를 포함한다. 주요 변경 사항은 빌드 환경과 일부 내부 로직에 한정되며, 외부 API(JSON‑RPC)에는 직접적인 변동이 없다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드에 직접적인 영향은 없으며, 메모리 할당 로직 변동이 스냅샷 생성 시 약간의 메모리 사용량 차이를 일으킬 수 있다. 대규 스냅샷을 자주 수행하는 환경이라면 재검증 권장.

## 운영 액션 아이템
- Docker 이미지 버전 업데이트(1.23-alpine) 적용
- CI 파이프라인에서 Go 1.23 버전으로 교체
- 노드 빌드 시 `go mod tidy` 및 테스트 실행
- 메모리 사용량 모니터링(스냅샷 시) 및 필요시 튜닝
- IsOnCurve 검증 로직이 정상 동작하는지 테스트(서명 검증 시나리오)

## 마이그레이션 체크리스트
- Go 1.23 환경에서 `go test ./...` 전체 테스트 실행
- Dockerfile 및 Dockerfile.alltools 업데이트 후 이미지 빌드 검증
- "build/checksums.txt"의 Go 1.23 체크섬스가 정상인지 확인
- 메모리 할당 로직(특히 memsize 관련) 테스트 케이스 검증
- IsOnCurve 로직이 기존 서명 검증에 미묘한 영향이 없는지 확인
- 아카이브 노드(스냅샷) 테스트 및 메모리 사용량 모니터링

## 위험/주의 사항
- Go 1.23으로의 업그레이드가 의존성 패키지와 빌드 스크립트에 미묘한 호환성 이슈를 일으킬 수 있다. 특히 Alpine 기반 Docker 이미지가 변경되었으며, 최신 Go 버전에서 발생할 수 있는 ABI/컴파일러 차이로 인한 런타임 이슈가 발생할 가능성.
- crypto/crypto.go 에서 추가된 `IsOnCurve` 체크는 내부적으로 서명 검증 로직에 영향을 주므로, 기존에 비활성화된 서명 검증이 활성화되면 일부 노드가 오래된 서명을 거부할 수 있다. 이는 RPC에서 `eth_sendRawTransaction` 같은 엔드포인트에 간접 영향을 줄 수 있다.
- memsize 삭제는 메모리 할당 로직에 미묘한 변동을 가져오며, 대규 모듈이 메모리 사용량에 민감한 경우(예: 대용량 스냅샷) 성능 이슈가 발생할 수 있다.

## 운영 메모
- 빌드 스크립트와 Docker 이미지가 Go 1.23으로 전환되었으므로, CI 파이프라인을 재구성하고 테스트를 재실행해야 한다.
- 메모리 사이즈 관련 삭제는 내부 메모리 할당 로직에 미묘한 변동을 가져오므로, 대용량 스냅샷 시 메모리 사용량이 변동될 수 있다.
- IsOnCurve 검증은 서명 검증 로직에 추가되었으나 외부 API에는 영향이 없으며, 노드가 오래된 서명을 거부할 가능성은 낮다.
