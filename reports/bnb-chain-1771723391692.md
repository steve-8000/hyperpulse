## 한줄 요약
버전 1.6.7 릴리즈는 주로 CI/워크플로우, ABI 코드베이스, 그리고 일부 내부 테스트 데이터 정리에 집중한 변경사항이다. 주요 기능적 변화는 없으며, API/JSON‑RPC 인터페이스에 직접적인 변경은 없다. 주요 포인트는 Go 1.24.x 버전 사용으로 인한 빌드 환경 조정과 일부 테스트/데이터 파일 삭제가 포함된다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 외부 API 엔드포인트/버전 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 비콘트롤 파일 삭제 및 테스트 데이터 정리

## 운영 액션 아이템
- Go 1.24.x 환경에서 빌드 테스트를 재실행
- CI 파이프라인에 `setup-go@v3` 로 교체된 스크립트 검증
- 테스트 데이터 파일 삭제에 대한 테스트 재실행(특히 `cmd/evm/testdata`)
- 스냅샷 스크립트(`cmd/geth/snapshot.go`)가 정상 동작하는지 확인
- RPC 엔드포인트에 대한 변경이 없으므로, 기존 클라이언트는 그대로 사용 가능

## 마이그레이션 체크리스트
- Go 버전(1.24.x) 로 빌드 환경 재설정
- CI 워크플로우 스크립트(`.github/workflows/*.yml`) 검증
- 테스트 데이터 파일 삭제에 대한 테스트 재실행
- 스냅샷/스토리지 관련 스크립트(`cmd/geth/snapshot.go`) 정상 동작 여부 확인
- RPC 엔드포인트 변동이 없으므로 클라이언트 코드에 대한 마이그레이션 필요 없음

## 위험/주의 사항
- 빌드 파이프라인에서 Go 1.24.x를 사용하도록 설정되었으나, 기존에 1.25.x 를 의존하는 패키지(예: `golangci-lint`, `actions/setup-go@v5`)가 1.25.x 전용으로 작성된 부분이 있다면 빌드 실패 가능
- 테스트 환경에서 특정 테스트가 실패할 수 있으나, 운영 노드에는 영향 없음

## 운영 메모
- 현재 커밋에서는 RPC/JSON‑RPC 인터페이스에 대한 직접적인 변경은 없으며, `cmd/geth` 의 내부 로직만 소폭 수정되었다. 따라서 기존 RPC 엔드포인트는 그대로 유지된다.
- `beacon/params/*` 에서 체크포인트 파일이 추가·수정되었으나, 이는 메인넷/시뇼폴리용 체크포인트에 대한 값만 변경되었으며, 아카이브 노드(스냅샷/스토리지)에는 영향이 없다.
