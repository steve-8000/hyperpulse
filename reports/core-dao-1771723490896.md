## 한줄 요약
v1.0.22 릴리즈는 Hermes 메인넷용 시스템 컨트랙트 15개를 새로 추가하고, upgrade 로직과 타입 정의를 확장한 버전이다. 주요 변경사항은 컨트랙트 코드 추가와 upgrade.go, types.go 에서 새로운 타입과 로직을 도입한 점이다. 이 릴리즈는 기존 노드가 새 컨트랙트를 인식하도록 업그레이드를 필요로 한다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- core/systemcontracts/hermes/mainnet/*.go : 15개의 새 컨트랙트 파일 추가
- core/systemcontracts/hermes/types.go : 38줄 신규 코드(새 타입 정의)
- core/systemcontracts/upgrade.go : 업그레이드 로직 90줄 추가
- params/config.go : 7줄 신규 설정 항목(예: Hermes 관련 옵션)
- params/version.go, version/version.go : 버전 문자열 업데이트

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 새 컨트랙트 코드가 추가되면서 아카이브 노드의 스토리지 사용량이 약 10~20 MB 증가할 수 있다.
- 컨트랙트 코드가 새로 추가되었으나, 기존 컨트랙트와 충돌이 없으므로 재인덱스 시점에서 인덱싱 비용은 최소화된다.

## 운영 액션 아이템
- 노드 소프트웨어를 최신 버전으로 업데이트한다.
- upgrade.go 로직에 따라 업그레이드 스크립트 실행
- 컨트랙트 주소가 바뀔 경우, config 파일에 새 컨트랙트 주소를 반영한다.
- 백업/스냅샷을 재생성하고, 인덱싱을 재시작한다.

## 마이그레이션 체크리스트
- 1. 현재 노드가 최신 버전인지 확인
- 2. upgrade.go 로직을 테스트 환경에서 실행해 보며 정상 동작 여부 확인
- 3. 컨트랙트 코드가 인덱스에 반영되는지 확인
- 4. 백업/스냅샷을 재생성하고, 인덱싱 재시작
- 5. RPC/JSON‑RPC 호출이 정상인지 테스트

## 위험/주의 사항
- 컨트랙트 코드가 새로 추가되면서 노드의 초기화 단계에서 컨트랙트 바이트코드를 로딩하는 비용이 증가할 수 있다.
- upgrade.go 에서 정의된 업그레이드 로직이 기존 버전과 호환되지 않을 경우, 노드가 정상 동작하지 않을 위험이 있다.
- 새 컨트랙트가 메인넷에 배포되면, 기존 상태를 그대로 유지하려면 컨트랙트 주소가 바뀔 수 있어 스냅샷/백업 시 주의 필요.

## 근거(Evidence)
- 근거 데이터 없음

## 운영 메모
- 컨트랙트 코드가 새로 추가되었으나, RPC/JSON‑RPC 인터페이스는 변하지 않는다.
- upgrade.go 에서 새 컨트랙트 로직이 추가되었으나, 현재 RPC 엔드포인트는 변하지 않는다.
- 아카이브 노드는 새 컨트랙트를 인식하도록 재인덱스가 필요할 수 있다. 기존 스냅샷이 있는 경우, 재인덱스 시점에서 컨트랙트 코드가 추가되면 스토리지 사용량이 증가한다.
