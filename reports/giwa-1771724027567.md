## 한줄 요약
본 릴리즈는 Dockerfile에서 베이스 이미지 버전을 bullseye → bookworm 으로 변경하고, Ubuntu 버전도 jammy → noble 로 업데이트한 단순 이미지 업그레이드이다. 핵심 소스코드는 변하지 않았으며, RPC/API/JSON‑RPC 인터페이스에는 직접적인 변경이 없다. 하지만 베이스 이미지가 바뀌면서 빌드 환경(라이브러리 버전, glibc 등)이 달라져서 빌드 타임과 실행 환경에 미묘한 영향이 있을 수 있다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드가 새로 빌드된 바이너리를 사용하도록 할 경우, 베이스 이미지 변경으로 인한 glibc/openssl 버전 차이로 인해 실행 시 경미한 충돌 가능성. 테스트 환경에서 재현 후 배포.

## 운영 액션 아이템
- 새 이미지 빌드 시 `docker build` 단계에서 베이스 이미지가 bookworm 으로 바뀐 것을 확인하고, 빌드 로그를 기록한다.
- 빌드된 바이너리의 `go version` 및 `glibc` 버전을 확인해 호환성 테스트를 진행한다.
- 아카이브 노드에 배포 시, 기존 이미지와 동일한 베이스 버전으로 재빌드 여부를 검토한다.

## 마이그레이션 체크리스트
- 베이스 이미지 버전(ubuntu:noble, golang:1.23.0-bookworm 등) 확인
- 빌드 시 의존성 패키지(예: libssl, glibc) 버전 차이 검토
- 빌드된 바이너리의 `go env` 및 `ldd -v` 결과를 기록
- 아카이브 노드에 배포 전, 테스트 환경에서 실행 및 RPC 동작 검증
- 배포 후 모니터링 지표(예: 메모리 사용량, CPU 점유) 기록

## 위험/주의 사항
- 베이스 이미지가 변경되면서 빌드 시 의존성 패키지 버전이 달라져, 이로 인한 빌드 실패 가능성
- Ubuntu:jammy → noble 로 변경되면서, 일부 패키지(예: libssl, glibc) 버전이 달라져 실행 시 동작 차이가 발생할 수 있음
- 빌드 도중에 사용되는 `just` 및 `cargo` 버전이 변경되면서, 빌드 스크립트가 의존성에 따라 달라질 수 있음

## 근거(Evidence)
- 근거 데이터 없음

## 운영 메모
- 베이스 이미지가 bookworm 으로 바뀌면서, 빌드 시 사용되는 패키지(예: libssl, glibc) 버전이 달라져 빌드 결과물에 미묘한 차이가 발생할 수 있다. 이는 테스트 환경에서 재현해 보아야 함.
- 이미지 빌드 시 `make` 단계에서 의존성 패키지가 변경되면, 실행 파일의 ABI가 달라질 수 있으나 현재 소스코드에는 변경이 없으므로 RPC/API는 그대로 유지된다.
- 아카이브 노드에 대한 직접적인 영향은 없으나, 빌드 시 사용되는 베이스 이미지가 바뀌면서 생성된 바이너리의 실행 환경이 달라져, 아카이브 노드가 동일 이미지에서 재배포될 경우 재시작 시 버전 충돌이 발생할 수 있다.
