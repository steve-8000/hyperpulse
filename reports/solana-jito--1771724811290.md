## 한줄 요약
본 릴리즈는 Solana v3.1.8 버전으로, 주로 내부 의존성 버전을 3.1.7 → 3.1.8 로 업데이트하고, Ledger‑Tool에서 스냅샷 생성 기능을 활성화하며, Entry 모듈에서 VersionedTransaction의 preallocation 한도를 상향시킨다. 주요 변경 사항은 API/JSON‑RPC 레벨에서 직접적인 인터페이스 변화가 없으나, 스냅샷 생성 로직과 메모리 할당 정책이 바뀌어 노드 운영에 영향을 미친다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 해당 사항 없음

## 운영 액션 아이템
- 노드 소프트웨어를 v3.1.8 로 업그레이드
- 스냅샷 생성 옵션(`--enable-snapshot-generation`) 을 활성화하고, 스냅샷 저장소를 충분히 확보
- 노드 메모리 사용량을 모니터링하고 필요 시 `max_preallocation_bytes` 를 조정
- 스냅샷 생성 주기를 재조정해 I/O 부하를 최소화

## 마이그레이션 체크리스트
- 코드베이스 의존성 버전 업데이트가 정상적으로 반영되었는지 확인
- `ledger-tool` 의 스냅샷 생성 플래그가 활성화되었는지 테스트
- 노드에서 `max_transaction_size` 혹은 `max_preallocation_bytes` 설정을 재검토
- 스냅샷 파일 크기와 저장소 용량을 모니터링
- 스냅샷 생성 주기 조정 테스트

## 위험/주의 사항
- 노드가 처리하는 트랜잭션 수가 늘어 메모리 사용량이 증가할 가능성
- 아카이브 노드가 스냅샷을 재생성할 때 I/O 부하 및 저장소 사용량 증가

## 근거(Evidence)
- 근거 데이터 없음

## 운영 메모
- 코드베이스 전역에서 `agave-*` 패키지 버전이 3.1.7 → 3.1.8 로 일괄 업데이트되었으며, 이는 내부 의존성 체인에 영향을 주지만 외부 API에는 직접적인 변동이 없다.
- 스냅샷 생성 기능이 활성화되었으나, 현재는 `ledger-tool` 내부에서만 동작하도록 설정되어 있어 RPC 레이어에는 영향이 없다.
- preallocation 한도 상향은 `entry/src/entry.rs` 에서 구현되었으며, 이는 트랜잭션 처리 시 메모리 할당 로직에만 영향을 미친다.
