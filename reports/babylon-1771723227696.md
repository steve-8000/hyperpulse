## 한줄 요약
v4.2.5 릴리즈는 주로 CI 워크플로우에 대한 권한 설정을 추가하고, 버전 표기만 업데이트된 소규한 변경이다. 주요 기능적 변화는 없으며, 운영자는 기존 설정을 그대로 유지하면서 추가된 권한이 적용되었는지 확인하면 된다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- CI 실행 시 보안 이벤트를 기록하고 토큰을 발급받는 기능이 활성화된다.

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 해당 사항 없음

## 운영 액션 아이템
- CI 워크플로우 파일(.github/workflows/ci.yml)에 추가된 권한을 반영하도록 배포
- 보안 이벤트 기록이 필요하다면 `security-events` 토큰을 발급받는 설정을 검토
- id-token 사용 시 토큰 발급 정책이 변경되었는지 확인

## 마이그레이션 체크리스트
- CI 워크플로우 파일에 추가된 권한이 정상적으로 적용되는지 테스트
- 보안 이벤트 기록 기능이 필요하다면 `security-events` 토큰을 발급받는지 확인
- id-token 사용 시 토큰이 정상적으로 발급되는지 검증

## 위험/주의 사항
- CI 워크플로우에 `security-events`와 `id-token` 권한이 추가되었으므로, 기존 워크플로우가 외부 서비스와 연동되는 경우 보안 설정을 재검토할 필요가 있다.

## 근거(Evidence)

## 운영 메모
- CI 설정에 보안 이벤트 기록과 토큰 발급 권한이 추가되었으나, 실제 API/JSON‑RPC 인터페이스에는 변화가 없다.
- 아카이브 노드에 대한 영향은 없으며, 기존 스크립트와 호환성 문제도 발생하지 않는다.
