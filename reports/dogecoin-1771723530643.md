## 한줄 요약
Dogecoin Core 1.14.9는 주로 보안·성능 개선과 AuxPoW(외부 PoW) 검증 로직에 대한 버그 픽스가 포함된 마이너버전이다. 주요 변경사항은 AuxPoW 검증 시 입력 존재 여부를 확인하고, 파트리얼 블록 처리 로직을 보다 견고하게 바꾸며, 일부 RPC/JSON‑RPC 인터페이스는 변동이 없으나 내부 로직 변경으로 인한 성능 개선이 반영된다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 운영 액션 아이템
- 노드 소프트웨어를 최신 버전(1.14.9) 으로 업데이트
- AuxPoW 검증 로직이 강화된 점을 감안해, 테스트넷에서 AuxPoW 블록을 수신 후 정상 동작 여부 확인
- 아카이브 노드가 오래된 파트리얼 블록을 재구동할 때 예외를 감지하면, `PartiallyDownloadedBlock` 로직을 재검토
- RPC 호출 빈도(특히 `getheaders`)를 모니터링해, 1.14.9에서 트래픽 감소가 반영되었으나 클라이언트에 따라 재조정 필요

## 마이그레이션 체크리스트
- 1. 소스코드(또는 패키지) 최신 버전으로 교체
- 2. AuxPoW 검증 로직이 정상 동작하는지 테스트넷에서 확인
- 3. 파트리얼 블록 재구동 시 예외가 발생하지 않는지 확인
- 4. `GetHeaders` 메시지 수량이 감소했는지 모니터링
- 5. RPC/JSON‑RPC 인터페이스가 변동이 없으므로, 별도 변경 필요 없음

## 위험/주의 사항
- 중간 단계에서 AuxPoW를 포함한 블록이 정상적으로 수신되지 않을 경우, 전체 체인에 대한 일시적 동결이 발생할 수 있다.
- 아카이브 노드에서 오래된 파트리얼 블록을 재구동할 때 예외 발생 가능성.

## 운영 메모
- 버그 픽스가 포함된 만큼, AuxPoW를 활용한 노드(특히 마이너)에서는 최신 버전으로 즉시 업데이트 권고.
- 아카이브 노드가 오래된 파트리얼 블록을 재구동할 때 예외 발생 가능성이 있으므로, `PartiallyDownloadedBlock` 로직을 재검토할 것.
- RPC/JSON‑RPC 인터페이스는 변동이 없으나, `GetHeaders` 메시지 수량 감소에 따라 일부 클라이언트가 `getheaders` 호출 빈도를 재조정할 필요성.
