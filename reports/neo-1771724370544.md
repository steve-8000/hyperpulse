## 한줄 요약
v3.9.0 릴리즈는 주로 내부 구조 정리와 보안/성능 개선을 목표로 한 대규모 변경이 포함되어 있다. 핵심은 RPC/JSON‑RPC 인터페이스에 대한 직접적인 변경이 적지만, 일부 엔드포인트와 정책 관련 파라미터가 조정되었다. 특히 `execFeeFactor` 와 `GetSignData` 가 새로 추가/변경되었으며, 백업/아카이브 노드에 대한 영향은 주로 스토리지 레이어와 메모리 캐시 동작이 바뀐 점에 있다. 이 릴리즈는 기존 플러그인/CLI를 제거하고, LevelDB 대신 메모리 스토리지 옵션을 기본으로 바꾸어 테스트 환경이 단순화되었다. 이로 인한 운영자는 노드 재배포 시 백업/아카이브 설정을 재검토해야 한다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- execFeeFactor 정책값 변경(FAUN)
- GetSignData 엔드포인트 추가
- LevelDB 스토리지 옵션 기본값 변경(메모리)
- CLI/플러그인 코드 삭제

## RPC/API 영향
- GetSignData

## 아카이브 노드 영향
- 레벨DB 스토리지 옵션이 기본이 아닌 경우, 메모리 스토리지 전환 시 데이터 일관성 이슈 가능

## 운영 액션 아이템
- 1. `execFeeFactor` 값이 변경된 점을 확인하고, 기존 컨트랙트/스크립트에 반영 여부 검토
- 2. `GetSignData` 엔드포인트를 호출하는 클라이언트가 타입 검증 로직을 추가
- 3. 레벨DB 스토리지 옵션이 비활성화되었으므로, 아카이브 노드가 메모리 스토리지 사용 시 설정 재검토
- 4. 백업/아카이브 노드에 대한 스냅샷 및 재시작 테스트 수행

## 마이그레이션 체크리스트
- 1. `execFeeFactor` 정책값이 변경되었는지 확인
- 2. 새로 추가된 `GetSignData` 엔드포인트가 정상 동작하는지 테스트
- 3. 레벨DB 스토리지 옵션이 비활성화되었는지 확인
- 4. 백업/아카이브 노드 설정(스토리지, 캐시) 재검토
- 5. JSON‑RPC 엔드포인트가 변동되지 않았는지 최종 검증
