## 한줄 요약
이번 릴리즈(1.12.20‑unstable → 1.12.20‑stable)는 Go 1.21 지원, p2p/Discover 로직 리팩터링, EIP‑4844(Blob) 지원 등 다수의 코드 변경을 포함한다. 주요 변경 사항은 RPC/JSON‑RPC 인터페이스 조정, 네트워크 프로토콜(Discover) 개선, 블록 헤더 검증 로직 업데이트이며, 이로 인한 운영 리스크와 마이그레이션 포인트를 정리한다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 외부 API 엔드포인트/버전 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- "over" RPC 메서드가 삭제되었습니다. 이로 인한 기존 클라이언트에서 해당 메서드를 호출하면 404/MethodNotFound 오류가 발생합니다. "RPCDiscover" 메서드도 제거되었습니다. 이 변경은 `cmd/ethclient` 내부 테스트에서 확인됩니다.

## 아카이브 노드 영향
- 1. `consensus/ethash`의 헤더 검증 로직이 업데이트되어, 스냅샷 인덱스가 오래된 경우 재인덱싱이 필요합니다. 2. EIP‑4844(Blob) 지원으로 인한 헤더 필드가 추가되어, 오래된 인덱스는 재구축이 필요합니다. 3. Discover 로직 변경으로 인한 노드간 연결 시 포트/IP 파싱이 바뀜. 오래된 Discover 클라이언트는 재설정 필요

## 운영 액션 아이템
- 1. Go 1.21 환경으로 빌드 2. RPC 메서드 삭제에 따른 클라이언트 코드 업데이트(예: `over` 혹은 `RPCDiscover` 호출 부분) 3. Discover 로직 변경에 따른 포트/IP 파싱 로직 검증 4. Archive‑Node 인덱스 재구축(특히 오래된 스냅샷) 5. EIP‑4844 헤더 필드 추가에 따른 인덱스 재구축

## 마이그레이션 체크리스트
- 1. Go 버전 1.21로 빌드 2. RPC 메서드(예: `over`, `RPCDiscover`)가 삭제된 부분을 클라이언트 코드에 반영 3. Discover 로직(예: `p2p/discover/v5wire`)이 바뀜 부분을 테스트 4. 스냅샷 인덱스가 오래된 경우 재인덱싱 5. EIP‑4844(Blob) 지원에 따른 헤더 필드가 추가된 경우 인덱스 재구축
