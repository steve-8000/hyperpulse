## 한줄 요약
Lotus v1.35.0은 주로 Ethereum RPC 호환성 개선과 CLI 기능 확장, OpenTelemetry 업데이트를 포함한다. 핵심 변경은 RPC 에러 코드 재배정(EExecutionReverted ↔ EActorNotFound)이며, 이는 클라이언트와 서버 간 시리얼라이즈 오류를 일으킬 수 있다. 또한 OpenRPC 문서 버전이 1.35.0으로 갱신되었다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 외부 API 엔드포인트/버전 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드가 RPC 에러 코드 변동에 의존하는 로직이 없으므로 직접적인 영향은 적음.
- OpenRPC 문서 버전 갱신이 아카이브 노드가 API 스키마를 재검증할 필요는 없으나, 문서 자동화 도구가 새 버전으로 재생성 해야 함.

## 운영 액션 아이템
- 1. 모든 노드(메인, 게이트웨이, 워커)에서 최신 1.35.0 버전으로 업그레이드.
- 2. RPC 클라이언트(예: lotus-api, lotus-client)에서 에러 코드 매핑을 새로 반영하도록 업데이트.
- 3. 테스트 환경에서 `errors.Is/As` 로직이 정상 동작하는지 검증.
- 4. OpenTelemetry 설정을 새 버전에 맞춰 재구성(필요 시).
- 5. FRC‑0102 서명 포장이 필요하면, wallet 관련 스크립트/스테이트를 재검증.

## 마이그레이션 체크리스트
- 1. 현재 운영 중인 노드가 1.35.0 버전으로 업그레이드 가능한지 확인.
- 2. RPC 에러 코드 재배정에 따른 시리얼라이즈 테스트(클라이언트/서버 모두).
- 3. OpenRPC 문서가 새 버전으로 재생성되었는지 확인.
- 4. OpenTelemetry 설정이 1.39.0에 맞게 업데이트되었는지 검증.
- 5. 새 서명 포장 기능이 필요하면, wallet 관련 테스트 케이스를 추가.

## 위험/주의 사항
- RPC 에러 코드 재배정으로 인한 시리얼라이즈 오류: 클라이언트가 EExecutionReverted(3)와 EActorNotFound(11)를 잘못 파싱하면 `errors.Is/As` 체크가 실패한다.
- 새로운 FRC‑0102 서명 포장이 도입되면서 wallet 관련 RPC 호출 시 입력 파라미터 검증 로직이 추가되어, 기존 스크립트가 실패할 가능성 존재.
- OpenTelemetry 버전 업그레이드로 로그 포맷이 바뀌서 기존 모니터링 스크립트가 예외를 발생시킬 수 있다.

## 운영 메모
- 에러 코드 재배정은 RPC 클라이언트가 1.34.x 버전과 혼용 시 에러 파싱이 잘못되므로, 업그레이드 전 테스트가 필수.
- FRC‑0102 서명 포장은 `wallet sign`/`verify` 명령에 새 옵션이 추가되었으나, 기존 스크립트가 서명 포장을 기대하지 않으면 실패할 수 있다.
- OpenTelemetry semconv 버전이 1.39.0 으로 바뀌서, 기존 트레이스 포맷이 변동될 수 있다.
