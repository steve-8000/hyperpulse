## 한줄 요약
3.1.1‑rc1 릴리즈는 주로 Batch 기능과 관련된 Feature 플래그를 비활성화하고, 버전 문자열을 3.1.0 → 3.1.1‑rc1 로 업데이트한 버전입니다. 주요 변경 사항은 features.macro 에서 Batch, BatchInnerSigs 플래그를 비활성화하고, BuildInfo.cpp 에서 버전 문자열을 수정한 것뿐입니다. 이로 인한 RPC/JSON‑RPC 인터페이스는 직접적으로 바뀌지 않으나, Batch 기능이 비활성화되면서 해당 기능을 호출하는 클라이언트는 동작하지 않을 수 있습니다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 주요 변경점 없음

## RPC/API 영향
- 해당 사항 없음

## 아카이브 노드 영향
- 아카이브 노드가 Batch 기능을 사용하지 않으므로, 별도 영향 없음. 단, Batch 관련 플래그가 비활성화되었으므로, 과거 데이터에 대한 Batch 처리 로직이 필요 없는 경우에는 무시해도 됨.

## 운영 액션 아이템
- 노드 소프트웨어를 최신 버전(3.1.1‑rc1) 으로 업그레이드
- Batch 기능이 비활성화된 것을 인지하고, 클라이언트가 Batch 호출을 사용하지 않도록 확인
- BatchInnerSigs 플래그를 Unsupported 로 설정한 점을 반영해 서명 검증 테스트 수행
- BuildInfo.cpp 의 버전 문자열이 3.1.1‑rc1 로 바뀐 것을 확인하고, 배포된 바이너리와 문서간 버전 일치 여부 검증

## 마이그레이션 체크리스트
- features.macro 에서 Batch, BatchInnerSigs 플래그가 비활성화 되었는지 확인
- Batch 기능을 활용하는 RPC 호출이 있는지 점검(예: /batch)
- Batch 관련 서명 검증 로직이 정상 동작하는지 테스트
- BuildInfo.cpp 의 버전 문자열이 3.1.1‑rc1 으로 설정되었는지 확인
- 배포된 바이너리와 문서간 버전 일치 여부 검증

## 위험/주의 사항
- Batch 기능이 비활성화되면서 기존에 Batch를 활용하던 RPC 호출(예: /batch) 이 실패할 가능성
- BatchInnerSigs 플래그가 Unsupported 로 전환되면서 서명 검증 로직이 변경될 수 있음
- 버전 문자열이 3.1.1‑rc1 로 변경되면서 배포된 바이너리와 문서간 버전 불일치 위험

## 근거(Evidence)
- 근거 데이터 없음

## 운영 메모
- Batch 기능이 비활성화되었으므로, 기존에 Batch를 활용한 RPC 호출은 동작하지 않을 수 있다. 이는 클라이언트 측에서 Batch 기능을 사용하고 있는 경우에 영향을 미친다.
- BatchInnerSigs 플래그가 Unsupported 로 전환되면서 서명 검증 로직이 변경될 수 있으므로, 서명 관련 테스트를 재검토할 필요가 있다.
- 버전 문자열이 변경되었으나, RPC/JSON‑RPC 인터페이스에는 직접적인 변화가 없으므로, API 레벨에서는 별도 변경이 없다.
