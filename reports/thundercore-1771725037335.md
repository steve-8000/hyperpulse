## 한줄 요약
ThunderCore R4.3(validator) 릴리즈는 기존 풀노드 설정에서 검증자용으로 전환된 점이 특징이다. 주요 변경사항은 컨테이너 이름·포트, keystore 마운트 추가, RPC 설정 삭제 및 Bidder 관련 파라미터 도입이다. 이로 인해 기존 RPC/JSON‑RPC 인터페이스가 비활성화될 가능성이 있으며, 아카이브 노드와의 호환성은 크게 변동되지 않는다. 다만, 새 필드(`bidder`, `metrics`) 도입으로 설정 파일을 재작성해야 한다.

## 운영자 중요 체크
- RPC/JSON-RPC 관련 변경 단서가 감지되었습니다.
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 1. docker‑compose 파일에서 `thunder-full` → `thunder-validator`, 포트 8545/8546 삭제, keystore 마운트 추가. 2. `override.yaml` 에서 RPC 섹션 삭제, 대신 `bidder` 파라미터 도입. 3. `isVoter: true` 로 고정, `loggingId` 주석 추가. 4. `metrics.address` 를 0.0.0.0:9201 로 고정. 5. `txLookupLimit` 32400 으로 설정.

## 위험/주의 사항
- 1. RPC/JSON‑RPC 인터페이스가 `override.yaml`에서 완전 삭제되었으며, docker‑compose에서도 8545/8546 포트가 제거됐다. 이는 외부에서 RPC 호출이 불가능해질 수 있다. 2. `isVoter`가 true 로 고정되었으나, 기존 설정에서 `isFullNode` 가 true 인 부분이 삭제돼서 노드가 완전한 풀노드가 아닌 ‘voter’ 모드로만 동작한다. 이는 아카이브 노드가 블록 데이터 수집에 필요한 RPC를 제공하지 못할 위험을 내포한다. 3. `bidder` 파라미터(`rpcUrl`, `rewardAddress`, `bidaddress`, `amount`) 가 새로 도입되었으나, 값이 미설정 시 bidding 로직에 오류가 발생할 수 있다. 4. `txLookupLimit` 가 0 → 32400 으로 바뀜. 이는 과거의 대량 트랜잭션 조회 시 성능 저하를 야기할 수.
