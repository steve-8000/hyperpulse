## 한줄 요약
Bitcoin Cash Node 29.0.0 릴리즈는 RPC 인터페이스와 노드 동작에 중대한 변화를 도입한다. 주요 변경사항은 per‑peer 트래픽 제한, 새 필드(permitbaremultisig, maxdatacarriersize) 추가, 그리고 getblock/getrawtransaction에 대한 패턴 필터 옵션이다. 이로 인해 기존 클라이언트와 스크립트는 새 파라미터를 인식해야 하며, 아카이브 노드 역시 체크포인트와 스토리지에 대한 재검증이 필요하다.

## 운영자 중요 체크
- 아카이브 노드 운영(보관/색인/동기화) 관련 변경 단서가 감지되었습니다.
- 노드 설정값/프로토콜 플래그 관련 변경 단서가 감지되었습니다.

## 주요 변경점
- 1. RPC: permitbaremultisig, maxdatacarriersize 필드 추가(9a751a5f0, b262ce3c2). 2. RPC: getblock/getrawtransaction 패턴 옵션 추가(71ea195ef). 3. CLI: -peerratelimit 도입(204b5e81). 4. CMake/OSX SDK 버전 업그레이드(0a4e074a3). 5. blockstorage: header hash 재계산 방지(6e2ed5e59). 6. 스크립트 최적화(int64_t 사용) 및 BigInt fallback(63ff0b788).

## RPC/API 영향
- permitbaremultisig field added to getmempoolinfo
- maxdatacarriersize field added to getmempoolinfo
- patterns flag added to getblock RPC
- patterns flag added to getrawtransaction RPC

## 아카이브 노드 영향
- 1. 체크포인트 업데이트(업그레이드 11)와 관련된 스크립트 검증 로직이 바뀜. 2. blockstorage 최적화는 header hash 재계산 방지로 인한 메모리/성능 개선. 아카이브 노드는 기존 스토리지 구조를 그대로 사용하므로 별도 영향 없음.

## 운영 액션 아이템
- 1. per‑peer traffic limit 옵션(-peerratelimit) 활성화 여부를 확인하고 필요 시 설정. 2. RPC 파라미터(permitbaremultisig, maxdatacarriersize)와 패턴 옵션을 인식하도록 모니터링 스크립트/클라이언트를 업데이트. 3. getblock/getrawtransaction 패턴 옵션이 활성화된 경우, 대용량 요청 시 성능 모니터링. 4. blockstorage 최적화가 적용된 경우, 스토리지에 대한 재검증(헤더 해시 재계산 방지).

## 마이그레이션 체크리스트
- 1. RPC 파라미터(permitbaremultisig, maxdatacarriersize)와 패턴 옵션을 인식하도록 클라이언트/스크립트를 업데이트. 2. per‑peer rate limit 설정을 검증하고 필요 시 재시작. 3. 체크포인트 업데이트(업그레이드 11)와 스크립트 검증 로직이 정상 동작하는지 테스트. 4. blockstorage 최적화가 적용된 경우, 스토리지 구조 변동이 없으므로 별도 영향 없음.

## 위험/주의 사항
- 1. RPC 파라미터 추가로 인한 호환성 이슈: permitbaremultisig, maxdatacarriersize 필드가 getmempoolinfo에 추가되었으나 기존 RPC 클라이언트는 이 필드를 무시할 수 있다. 2. getblock/getrawtransaction에 패턴 옵션이 추가되면서, 대용량 데이터 요청 시 필터링 로직이 필요해 성능에 영향. 3. per‑peer traffic limit 옵션(-peerratelimit) 도입으로 기존 네트워크 설정이 변경되면 노드가 재시작 필요. 4. 체크포인트 업데이트(업그레이드 11)와 관련된 스크립트 검증 로직이 바뀜. 5. blockstorage에서 header hash 재계산 방지로 인한 메모리/성능 개선이 있지만, 스토리지 구조가 변하지 않음.

## 운영 메모
- 1. RPC 파라미터 추가에 따라 기존 스크립트/클라이언트는 새 필드 무시 가능. 2. 패턴 옵션은 대용량 요청 시 필터링 로직이 필요해 성능에 영향. 3. per‑peer traffic limit 옵션은 노드 재시작 필요. 4. blockstorage 최적화는 스토리지에 영향 없음. 5. 체크포인트 업데이트와 업그레이드 11 전환이 정상 동작하는지 확인 필요.
